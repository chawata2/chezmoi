# Obsidian Add スキル

Obsidian vault に知見を記録する。既存ノートへの追記か新規作成かを判断し、ユーザーの確認を取ってから書き込む。

## 設定

- **Vault**: `~/Documents/vault`
- **Notesディレクトリ**: `~/Documents/vault/Notes/`
- **テンプレート**: `~/Documents/vault/templates/Notes.md`
- **ルール参照**: `~/Documents/vault/CLAUDE.md`（毎回必ず読む）

---

## ワークフロー

### Step 1: ルールを読む

最初に必ず `~/Documents/vault/CLAUDE.md` を読む。文体・タグ規則・フロントマター仕様などがここに書いてある。これを守らないと、ノートが他のノートと一貫性を失う。

### Step 2: 既存ノートを調査する

Notes/ ディレクトリの既存ノートを2つのアプローチで調査し、「タイトルが同一または非常に近いノート」を探す。

**アプローチ1: キーワード検索**

記録する内容からキーワードを抽出して rg で検索する。
```
rg -il "<キーワード>" ~/Documents/vault/Notes/
```

**アプローチ2: タグ検索**

1. Notes/ 内の全ノートからタグ一覧を抽出する。インライン形式・複数行形式の両方に対応したスクリプトを使う。
   ```bash
   python3 <skill-base-dir>/scripts/extract_tags.py
   ```
   ※ `<skill-base-dir>` はスキルコンテキスト冒頭の "Base directory for this skill:" に示されるパス。
2. 今回の内容に関連しそうなタグを特定する
3. 該当タグを持つノートを探す
   ```
   rg -l "タグ名" ~/Documents/vault/Notes/
   ```

両アプローチの結果を合わせて候補を洗い出し、タイトルを確認する。

**追記候補の判断基準：**
- タイトルが**同一**または**ほぼ同じトピック**を指している場合のみ追記を推奨
- トピックが少し違う、角度が違う、詳細が違う → 新規作成を推奨

例：
- 「fish shellの設定」→ 既存の「fish.md」があれば追記候補
- 「fishのプロンプトカスタマイズ」→ 「fish.md」があっても内容次第で新規候補

### Step 3: ユーザーに選択肢を提示する

`AskUserQuestion` を使って確認する。

**追記候補がある場合の質問例：**
```
「<既存ノートタイトル>」に追記することを推奨します。

1. 追記する（推奨）
2. 新しいノートを作成する
3. 両方確認してから決める

どうしますか？
```

**追記候補がない場合の質問例：**
```
類似ノートは見つかりませんでした。新しいノートを作成します。

タイトル案：<AIが提案するタイトル>

1. このタイトルで作成する（推奨）
2. タイトルを変更する

どうしますか？
```

### Step 4: 変更内容を示してから確認する

実際に書き込む前に、変更内容をユーザーに見せる。

**追記の場合：**
```
## 追記する内容

**対象ファイル**: Notes/fish.md

---（追記内容ここから）---

## fishのエイリアス設定

...

---（ここまで）---

この内容を追記してよいですか？（はい/いいえ）
```

**新規作成の場合：**
```
## 作成するノート

**ファイル名**: Notes/fish プロンプトカスタマイズ.md

---（内容ここから）---

---
type: notes
repo:
tags: [fish]
---
## Overview

...

---（ここまで）---

この内容で作成してよいですか？（はい/いいえ）
```

確認にも `AskUserQuestion` を使う。

### Step 5: 書き込む

ユーザーがOKを出したら、実際にファイルを書き込む。

## ノートのフォーマット規則

`~/Documents/vault/CLAUDE.md` に従うこと。主な規則：

### フロントマター（Notes）
```yaml
---
type: notes
repo: リポジトリ名（該当する場合、なければ空欄）
tags: [タグ1, タグ2]
---
```

### タグ規則
- 既存タグの中から適切なものを使う（CLAUDE.mdに一覧あり）
- 既存タグにない新しいタグを使いたい場合は、`AskUserQuestion` でユーザーに確認してから追加する

### 文章スタイル
- 常体（だ・である）＋命令形（〜する。〜を実行する。）
- 一文70〜80文字目安
- 一文一義
- できるだけ具体的・肯定形で書く

### セクション構成（新規作成時）
- `## Overview`（必須）
- その後は内容に応じて自由に構成

### タイトル（ファイル名）
- 日本語のトピック名
- AIが内容から自動生成する（ユーザーの承認は不要、指摘があれば変える）
